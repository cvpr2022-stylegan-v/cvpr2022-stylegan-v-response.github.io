Content/motion decomposition

Content/motion decomposition for FaceForensics 256x256 (left) and SkyTimelapse 256x256 (right). Each row is a different content code, while each column is a different sequence of motion noise codes. Note that our method captures temporal patterns not only in terms of motion, but also appearance changes, like time of day.

Random very long videos generated with our model

We provide very long videos generated with our model (40 seconds each). While SkyTimelapse and RainbowJelly experience motion repetition to some extent, FaceForensics does not exhibit such artifacts. The reason is that our generator tries to reuse content as much as possible, because (as we believe) it exhausts the amount of information stored in a content latent code (since we generate very long videos). This shows the limitations of the classical paradigm of content/motion decomposition.

Illustration of the synthesis process

Illustration of the synthesis process of a video in timesteps t′,t″ and t‴ — during training, we sample 3 random timesteps per video, while at test-time we sample timesteps as a discrete grid with a predefined resolution (depending on the desired frame-rate). To generate a video at the given timesteps, we first sample a content noise vector zc and pass it through the Content Mapping network Fc to obtain content latent code w. Then, we sample motion noise vectors zt0m,zt1m,...,ztnm and pass them through the Motion Mapping Network Fm (with our acyclic positional encoding layer on top of it) to obtain continuous motion codes v(t). We then synthesize frames independently from the given codes with Synthesis Network S.

We use Synthesis Network S from StyleGAN2 without minimal changes: the only change is the concatenation of a (tiled to 4×4) motion code vt∈R512 to the learnable constant input tensor of size 4×4×512. SynthesisBlock is inhertied from StyleGAN2 without any changes at all: it's a block which takes the activations tensor (originally initialized with the learnable constant tensor) and the image tensor (originally initilized with zeros for the 4×4×512-resolution block) from the previous layer and transforms them with ModConv2d layers and skip connections (see the scheme). ModConv2d is a convolutional layer, which parameters θ∈Rcout×cin×kh×kw are multiplied by a style vector s=A(w)∈Rcin (obtained through the affine transformation A of the latent code w∈R512) across cin-axis (see StyleGAN2 for more details).

MTurk worker's interface

This is the interface which was used to conduct human evaluation experiments on Amazon Mechanical Turk. Each worker was shown a pair of random videos in random order and was asked to select the more realistic one. In total, there were 256 videos used from each method (randomly sampled). For comparison with MoCoGAN-HD, MoCoGAN+SG2 and DIGAN, we used short videos (since we wanted to show that they correlate properly with our FVD16 implementation). For comparison with LSTM-codes, we used long (128-frames-long) videos, since we wanted to compare the generated motions, which are easier to grasp on long videos.
